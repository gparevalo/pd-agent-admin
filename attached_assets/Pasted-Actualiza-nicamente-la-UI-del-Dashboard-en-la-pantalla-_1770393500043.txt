## 🧩 ESTRUCTURA DEL TAB "INTEGRACIONES"

El tab "Integraciones" debe tener dos secciones principales:

### 1️⃣ Integraciones del Agente
(usan tablas existentes: whatsapp_instances, n8n_workflows, agent_webhooks)

Cards:

#### 🟢 WhatsApp (Evolution API)

Mostrar una Card con:
- Ícono de WhatsApp
- Título: "WhatsApp Evolution API"
- Subtítulo: "Canal de mensajería conectado al agente"
- Estado de conexión (badge: Connected / Disconnected)
- Número conectado (si existe)
- Botones:
  - "Conectar WhatsApp"
  - "Ver QR"
  - "Desconectar"

Estados visuales:
- Loader mientras se genera QR
- Badge verde si conectado
- Badge rojo si desconectado
- Toasts: success / error

Al hacer clic en "Conectar" o "Ver QR":
Abrir un Modal con:

Modal WhatsApp:
- Título: "Conectar WhatsApp"
- Fondo blanco
- QR centrado
- Texto: "Escanea este código QR desde WhatsApp"
- Indicador de estado en tiempo real (conectando / conectado)
- Animaciones suaves (framer-motion)

---

#### 🟣 n8n Workflow

Mostrar una Card con:
- Ícono n8n
- Título: "n8n Workflow"
- Campos visibles:
  - Workflow Name
  - Workflow ID
  - Webhook URL
  - Estado (active / inactive)
- Botón: "Regenerar Workflow"

Modal n8n:
Al hacer clic en "Ver flujo":
Mostrar un modal con:
- Diagrama visual del flujo:
  WhatsApp → Webhook → n8n → Agente → Respuesta
- Texto explicativo del flujo
- Estilo visual tipo diagrama simple

---

### 2️⃣ Integraciones de la Empresa
(usan tabla: company_integrations)

#### 🔵 Go High Level (OAuth)

Mostrar Card con:
- Ícono GoHighLevel
- Título: "Go High Level"
- Subtítulo: "CRM conectado a la empresa"
- Estado: Connected / Disconnected
- Mostrar:
  - client_id (parcial oculto)
  - scope
- Botones:
  - "Conectar Go High Level"
  - "Desconectar"

Scope por defecto:
locations.readonly contacts.readonly contacts.write opportunities.readonly opportunities.write users.readonly calendars.readonly calendars.write calendars/events.write locations/customFields.readonly locations/customFields.write

Modal Go High Level:
Mostrar flujo OAuth visual:

Empresa → OAuth → Go High Level → Tokens → Plataforma

Campos en modal:
- Client ID
- Client Secret (input password)
- Scope (readonly prellenado)
- Botón "Autorizar"
